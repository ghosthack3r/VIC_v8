# Skill Pressure-Test Notes

Date: 2026-05-03

These notes summarize baseline mistakes captured before writing the new skills.

## Copy-First Development Baseline

Common rationalizations:

- "It is only a one-line fix."
- "Git can revert it."
- "The command edits files, but I did not manually edit them."
- "I need to inspect by trying small edits."
- "A faithful copy needs everything, including secrets."
- "The user wants autonomous improvements, so I should move fast."

Skill response: require a `codexEdit_` working copy before autonomous mutations and define default exclusions.

## Personal Project Manager Baseline

Common rationalizations:

- "I will just keep track as we go."
- "The conversation has enough context."
- "The subagent reports speak for themselves."
- "They said it was finished."
- "A dashboard will make this manageable."
- "This should be quick."

Skill response: maintain durable artifacts, status vocabulary, verification evidence, and subagent reconciliation.

## VIC/Aura Priority Baseline

Common rationalizations:

- "I should understand the whole repo before touching anything."
- "The UI is what the user will see first."
- "Tests can substitute for actually launching it."
- "Everything is equally important because it is all connected."
- "No test is meaningful without all devices connected."
- "I will compare all snapshots before deciding."

Skill response: use a driving-readiness priority ladder and bring-up loop.

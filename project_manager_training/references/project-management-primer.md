# Project Management Primer For Codex

Date: 2026-05-03

## Working Principles

Project management for this workspace should optimize for value, safety, clarity, and working increments.

PMI's principle set emphasizes stewardship, team collaboration, stakeholder engagement, value, systems thinking, leadership, tailoring, quality, complexity, risk, adaptability, and change. For this workspace, that translates into:

- Own the workspace carefully; do not hide risk or mutate originals casually.
- Keep the user and the user's near-term travel needs visible in every prioritization call.
- Prefer small working increments over large speculative plans.
- Tailor the process to the project instead of forcing ceremony.
- Treat secrets, safety-critical vehicle behavior, and build reliability as project risks, not side notes.

## Agile/Scrum Takeaways

The Agile Manifesto favors people, working software, collaboration, and responsiveness. The Scrum Guide frames Scrum as a lightweight way to generate value through adaptive solutions for complex problems.

For this workspace:

- The backlog should stay ordered by value and risk.
- A task is not done until its verification evidence is recorded.
- Review and adapt after each meaningful increment.
- Keep reports short enough that the user can inspect them quickly.
- Use "Definition of Done" per task: changed files, verification commands, known gaps, and next action.

## Codex Operating Loop

1. Orient: read the active `project_management` files and project instructions.
2. Protect: confirm the work is inside a `codexEdit_` copy before making edits.
3. Prioritize: pick the smallest task that advances the current project goal.
4. Execute: change only the files needed for that task.
5. Verify: run the command or inspection that proves the result.
6. Report: update status, risks, and next actions.

## What To Avoid

- Editing original folders because the change looks small.
- Treating build output or dependency installation as harmless when it mutates files.
- Starting feature work before credential, install, and boot risks are visible.
- Creating reports so detailed that the user stops reading them.
- Delegating vague tasks to subagents without ownership boundaries.
